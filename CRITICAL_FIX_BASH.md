# 🔴 关键修复: 强制使用 bash grep

## 问题根因（日志 20260327_111352_770.json）

模型在扫描任务中：
1. ❌ 使用 list_dir 列出目录
2. ❌ 使用 read_file 逐个读取文件
3. ❌ 看到内容太长后说"无法处理"
4. ❌ 尝试输出Python代码（不是工具调用）
5. ❌ **从未调用 bash grep**

## 核心问题

虽然添加了bash示例，但模型优先看到read_file/list_dir，习惯性使用这些工具。

## 最终修复

### 1. 工具列表重排序 ✅
```python
可用工具：
- bash: 执行 shell 命令（grep, find, awk 等，批量处理时优先使用）← 放第一位
- read_file: 读取单个文件内容
- write_file: 写入文件
- edit_file: 精确编辑文件
- list_dir: 列出目录内容
```

### 2. 新增工具选择策略 ✅
```
【工具选择策略】
- 批量扫描/搜索 → 优先用 bash (grep, find)
- 单个文件读取 → 用 read_file
- 目录浏览 → 用 list_dir
```

### 3. 示例重排序 ✅
```
【示例 1】扫描代码结构（最重要！批量任务必须用bash）← 放第一位
用户: 扫描 core 目录，找出所有类和方法
助手: bash
{"command": "grep -rn '^class \\|^def ' core/"}
```

### 4. 强化禁止行为 ✅
```
【重要提示】
- ⚠️ 扫描/搜索多个文件时，必须用 bash grep，禁止用 list_dir + read_file 逐个读取
- 不要输出Python代码示例，直接调用工具完成任务
```
