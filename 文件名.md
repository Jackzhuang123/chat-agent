# API 文档

## core 模块

### core/tool_learner

- **描述**: 工具学习模块 - 基于任务类型和历史自动选择工具

- **特性**:
  - 任务分类（文件操作、代码分析、系统命令等）
  - 工具成功率统计
  - 基于上下文的工具推荐
  - 持久化学习结果

- **类和方法**:
  - ToolLearner

### core/vector_memory

- **描述**: 向量记忆模块 - 使用 embedding 存储和检索历史对话

- **特性**:
  - 使用简单的 TF-IDF 作为 embedding（无需外部模型）
  - 支持基于嵌入的语义相似度检索
  - 持久化存储到硬盘
  - 自动去重

- **类和方法**:
  - VectorMemory

### core/agent_middlewares

- **描述**: Agent 中间件模块 - 实现 Middleware Chain 设计模式

- **特性**:
  - 所有中间件遵循 AgentMiddleware 接口
  - 可自由组合中间件以添加 QwenAgentFramework 功能
  - 每个中间件负责：
    - before_model: 模型调用前修改消息列表（添加上下文等）
    - after_model: 模型返回后处理文本
    - after_tool_call: 工具调用后标准化结果
    - after_run: 会话结束时处理

- **类和方法**:
  - AgentMiddleware

### core/agent_skills

- **描述**: Agent 技能系统 - 知识外部化

- **特性**:
  - 核心概念: 知识外部化
  - 与工具的不同:
    - 工具 = 模型能做什么 (bash, read_file, etc.)
    - 技能 = 模型知道怎么做 (PDF 处理, MCP 开发, 代码审查等)
  - 技能允许按需加载领域知识，无需重新训练模型。

- **类和方法**:
  - SkillManager

### core/agent_tools

- **描述**: Agent 工具系统 - 为 Qwen 模型提供工具支持

- **特性**:
  - 支持多种工具，如 bash, read_file, write_file, edit_file 等
  - 可配置工作目录
  - 可启用或禁用 bash 工具
  - 提供工具描述和输入输出规范

- **类和方法**:
  - ToolExecutor

### core/mode_router

- **描述**: 模式路由器 - 自动识别用户意图并切换模式

- **特性**:
  - 支持的模型:
    - chat: 纯对话模型（聊天、问答等）
    - tools: 工具模型（文件操作、代码分析、命令执行等）
    - skills: 技能模型（使用外部知识库）
    - hybrid: 混合模型（技能 + 工具）
    - plan: 计划模型（复杂任务分解）
    - multi_agent: 多 Agent 模式（需要规则-执行-检查）
    - streaming: 流模式（实时更新）
  - 自动识别用户意图并切换模型

- **类和方法**:
  - ModeRouter

### core/multi_agent

- **描述**: 多 Agent 协作模块 - Planner + Executor + Reviewer 架构

- **特性**:
  - 支持复杂任务分解和执行
  - 规则-执行-检查流程
  - 可配置工具集
  - 支持多 Agent 协作

- **类和方法**:
  - PlannerAgent
