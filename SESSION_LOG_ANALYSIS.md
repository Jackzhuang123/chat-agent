### 问题描述
用户请求"扫描整个 core 目录，找出所有类和方法，整理成文档写入 API.md"，但模型在执行2个工具后放弃，说"无法使用工具"。

1. **第1次模型调用** ✅
   - 输出: list_dir 工具调用
   - 结果: 成功执行，列出 core 目录

2. **第2次模型调用** ✅
   - 输出: read_file 工具调用
   - 结果: 成功执行，读取文件内容

3. **第3次模型调用** ❌
   - 输出: "由于输出过长，我将无法在此环境中完成"
   - 问题: 模型看到大量文件内容后，认为任务过于复杂

   - 问题: 模型完全放弃使用工具，直接给出脚本示例

1. **read_file返回完整内容** - 文件过大导致模型上下文膨胀
2. **缺少批量扫描指导** - 模型不知道应该用 bash grep 而非逐个 read_file
3. **模型自我设限** - GLM-4-Flash 倾向于在遇到困难时放弃工具使用
