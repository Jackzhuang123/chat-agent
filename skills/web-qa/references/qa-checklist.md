# Web QA 检查清单

移植自 gstack review/checklist.md 思路，针对 Web QA 场景定制。

## 部署验证（每次部署必检）

- [ ] 首页正常加载（HTTP 200，无 500 错误）
- [ ] 关键路由可访问（/login, /dashboard, /api/health）
- [ ] 无控制台 JS 错误
- [ ] 无失败网络请求（尤其是 API 调用）
- [ ] 标题/Logo 正确显示
- [ ] 响应时间 < 3秒（首次加载）

## 核心用户流程

- [ ] 注册流程（含验证邮件/手机号）
- [ ] 登录流程（正确凭据 + 错误凭据）
- [ ] 登出流程
- [ ] 密码重置流程
- [ ] 核心业务流程（产品/服务特定）

## 表单测试

- [ ] 空提交显示验证错误
- [ ] 必填字段标注清晰
- [ ] 错误信息具体且有帮助
- [ ] 成功提交后正确重定向/提示
- [ ] XSS 输入不导致脚本执行

## 响应式测试

- [ ] 移动端 375px（iPhone SE 尺寸）
- [ ] 平板 768px（iPad 尺寸）
- [ ] 桌面 1280px（标准宽度）
- [ ] 无横向滚动条（移动端）
- [ ] 导航菜单在移动端可用

## 可访问性（基础）

- [ ] 图片有 alt 属性
- [ ] 按钮有可识别文字或 aria-label
- [ ] 表单字段有关联 label
- [ ] 焦点状态可见（键盘导航）
- [ ] 对比度足够（WCAG AA）

## 性能基准

- [ ] 首次内容绘制 (FCP) < 1.8s
- [ ] 页面总大小 < 5MB
- [ ] 图片已压缩
- [ ] 无阻塞渲染资源
