# 代码审查技能

## 系统化审查流程

### 1. 代码结构 (10%)
- [ ] 函数长度合理 (< 50 行)
- [ ] 类职责单一
- [ ] 模块化设计

### 2. 命名和可读性 (20%)
- [ ] 变量名清晰
- [ ] 函数名描述性强
- [ ] 代码注释完整

### 3. 错误处理 (15%)
- [ ] 异常捕获适当
- [ ] 错误消息有用
- [ ] 边界条件处理

### 4. 性能 (15%)
- [ ] 算法复杂度合理
- [ ] 无明显性能问题
- [ ] 资源释放正确

### 5. 安全性 (20%)
- [ ] 输入验证
- [ ] SQL 注入防护
- [ ] 无硬编码密钥

### 6. 测试 (20%)
- [ ] 单元测试覆盖
- [ ] 边界测试
- [ ] 错误情况测试

## Python 特定检查

- 遵循 PEP 8 风格
- 类型提示完整
- 文档字符串存在

## 审查评语模板

```
总体评价: [好/待改进/需要重审]
关键问题: [列表]
改进建议: [列表]
建议批准: [是/否]
```
